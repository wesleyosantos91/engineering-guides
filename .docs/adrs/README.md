# Architecture Decision Records (ADRs) — Guia Completo

> **Objetivo deste guia:** Servir como referência completa sobre **Architecture Decision Records (ADRs)** — fundamentos, escrita, governança em escala e templates prontos, otimizada para uso como **base de conhecimento para assistentes de código (Copilot/AI)** e consulta humana.
>
> **Público-alvo:** Engenheiros de software, arquitetos e tech leads que desejam documentar decisões arquiteturais de forma estruturada e rastreável.

---

## Sumário

| Documento | Descrição |
|-----------|-----------|
| [01-adr-foundations](01-adr-foundations.md) | Fundamentos de ADRs — o que são, por que existem, quando usar, anatomia, tipos de decisões e ciclo de vida |
| [02-adr-writing-guide](02-adr-writing-guide.md) | Guia prático de escrita — processo de identificação, argumentação, review e integração com workflow de engenharia |
| [03-adr-governance-at-scale](03-adr-governance-at-scale.md) | Governança em escala — como escalar ADRs de um time para centenas, discovery, métricas, evolução e cultura |
| [04-adr-templates-examples](04-adr-templates-examples.md) | Templates prontos e exemplos completos — MADR, lightweight, exemplos por tipo de decisão (technology choice, integration, data, infra, security) |
